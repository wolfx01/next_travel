## 5. Application Pages & Structure
**Total Active Pages:** 7

The application consists of the following key routes, each serving a distinct user journey:

| Page Name | Route URL | Role & Functionality |
| :--- | :--- | :--- |
| **Home** | `/` | **Landing Page**: Features a video hero, "Beyond Borders" introduction, and showcases top destinations to engage users immediately. |
| **Countries** | `/countries` | **Exploration Hub**: Displays a comprehensive grid of countries. Allows users to browse destinations by nation. |
| **Places** | `/places` | **Discovery Engine**: detailed list of cities and attractions. Includes filtering by country and search functionality. |
| **Place Details** | `/places/[id]` | **Conversion Page**: The core transactional page. Displays deep details (population, currency), an image gallery, user reviews/comments, and the **Aviasales Flight Widget** for booking. |
| **About** | `/about` | **Brand Story**: informational page describing the "Explo" mission and team values. |
| **Login** | `/login` | **Authentication**: Secure entry point for returning users using JWT-based auth. |
| **Register** | `/register` | **Onboarding**: Signup form for new users, securely hashing passwords and storing profiles in MongoDB. |
